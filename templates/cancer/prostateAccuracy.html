                                                                                                                  Support Vector Machine


It is widely used model for classification as it produces significant accuracy with less computation power. The objective of the support vector machine algorithm is to find a hyper plane in an N-dimensional space (N -the number of features) that distinctly classifies the data points for a given data set.

Our objective is to find a plane that has the maximum margin that is the maximum distance between data points of both classes. Maximizing the margin distance makes sure that in future prediction not many data points will be wrongly classified.

Hyperplanes are decision boundaries that help to classify the data points. Data points on either side of the hyperplane are considered to belong to different classes. Also, the dimension of the hyperplane depends upon the number of features.

Support vectors are data points that are closer to the hyperplane and helps in choosing the decision boundary. Using these support vectors, we maximize the margin of the classifier. These are the points that help us build our SVM for the best or say accurate prediction of the test data set.

                                                                                                          Decision Tree Classifier

Decision Trees (DTs) are a non-parametric supervised learning method used for classification and regression. The goal is to create a model that predicts the value of a target variable by learning simple decision rules inferred from the data features. It basically applies some if-then-else decision rules set on the given training data set and predict the target or class label to which it might belong. The deeper the tree, the more complex the decision rules and the fitter the model. There are various types of decision trees such as ID3, C4.5, CART etc and these decision tress uses different split techniques such as :

1. Information gain:

The attribute with the highest information gain is selected first as the root node and so on goes the further selection procedure. It is based on the concept of entropy, which is the degree of uncertainty, impurity or disorder.

2. Gini index:

The attribute with least gini index is selected first as the root node because it measures the degree or probability of a particular variable being wrongly classified when it is randomly chosen.

                                                                                                                       Logistic Regression

Logistic regression is another technique borrowed by machine learning from the field of statistics. It is the go-to method for binary classification problems (problems with two class values). Logistic regression is named for the function used at the core of the method, the logistic function. The logistic function, also called as the sigmoid function. It’s an S-shaped curve that can take any real-valued number and map it into a value between 0 and 1, but never exactly at those limits.

1 / (1 + e^-value), where e is the base of the natural logarithms and value is the actual numerical value that you want to transform.

Logistic regression uses an equation as the representation, very much like linear regression. Input values (x) are combined linearly using weights or coefficient values to predict an output value (y). A key difference from linear regression is that the output value being modeled is a binary value (0 or 1) rather than a numeric value.


Below is an example logistic regression equation:

                            y = e^(b0 + b1*x) / (1 + e^(b0 + b1*x))

                


Where y is the predicted output, b0 is the bias or intercept term and b1 is the coefficient for the single input value (x). Each column in your input data has an associated b that must be learned from your training data. The actual representation of the model that you would store in memory is the coefficients in the equation.

                                                                                                                 Random Forest Classifier


Random forest, consists of a large number of individual decision trees that operate as an ensemble. Here ensemble means a collection or grouping of various decision tress rather than one decision tree, hence the name random forest. The training data is fed to the classifier, which then creates a bootstrap data, by randomly selecting the data and creates various decision trees.

So whenever a class needs to be predicted, each individual decision tree spits out a class label and then the class label with the majority is decided as the final predicted class. This is a lot better classifier than decision tree because there is less chance of inaccuracy and error as in a decision tree.

This classifier allows duplicity while selection of data for the bootstrap data which increases the randomization factor for that particular decision tree. A random forest also uses the same splitting techniques as that of decision tree and we can have numerous number of decision tress for better performance.

                                                                                                                   Naive Bayes Classifier

It works on the Bayes Theorem of probability to predict the class of an unknown data set and their data points. It is best suited for problems involving huge datasets with hug rows and less number of columns that is features. It is a classification technique based on Bayes Theroem with an assumption of independence among predictors. In simple terms, a Naive Bayes classifier assumes that the presence of a particular feature in a class is unrelated to the presence of any other feature. Hence the name “NAÏVE”.

                                                         


The steps to perform the Naïve Bayes classification:

 Convert the data set into a frequency table.

 Create Likelihood table by finding the probabilities of classes and the tuples to be classified/predicted etc.

 Calculate the posterior probability with the help of the likelihood table.

 Select the class with the highest prior probability and that will we the label for that test sample.
